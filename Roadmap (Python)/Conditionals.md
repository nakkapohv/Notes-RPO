#Conditionals #if #elif #else 
### Введение 

По умолчанию программы выполняются строго последовательно, шаг за шагом. Однако для решения реальных задач программе необходимо уметь пропускать команды, повторять их или выбирать альтернативные пути развития событий. За порядок выполнения кода отвечает поток управления, который регулируется специальными управляющими конструкциями. Базовой такой конструкцией в Python является условный оператор `if`, позволяющий программе принимать решения и выполнять код только в том случае, если определенное условие выполняется.
### Синтаксис условного оператора

В своей простейшей форме оператор записывается как `if ...:` . В отличие от многих других языков, в Python это выражение не нужно заключать в круглые скобки, но двоеточие в конце строки ставится обязательно. Если выражение оценивается как истинное, программа выполняет идущую следом инструкцию с отступом; если как ложное — инструкция полностью пропускается.
### Группировка кода и правило отступов

Часто при выполнении одного условия требуется запустить не одну, а сразу несколько команд. В Python для определения таких блоков используется так называемое «правило вне игры» - жесткая система отступов. Все идущие друг за другом строки кода, имеющие одинаковый уровень отступа, интерпретатор считает единым логическим блоком. Если условие `if` истинно, выполнятся все строки блока, если ложно — они все будут проигнорированы. Для обозначения конца блока кода в Python не используются закрывающие скобки или специальные символы. Конец логического блока наступает автоматически там, где появляется строка с меньшим отступом. При этом блоки можно вкладывать друг в друга на любую глубину: каждый новый уровень отступа создает внутренний блок (например, дополнительную проверку `if` внутри уже сработавшего `if`), а уменьшение отступа возвращает выполнение на уровень выше. Это делает структуру программы интуитивно понятной.
### Elif и Else

Простой оператор `if` выполняет код только тогда, когда условие истинно, и ничего не делает, если оно ложно. Для обработки альтернативных сценариев используются операторы `else` и `elif`. Оператор `else` (иначе) не требует никаких условий и ставится в конце: его блок кода срабатывает автоматически, если основная проверка в `if` провалилась. Если же нужно по очереди проверить несколько разных условий, применяется оператор `elif` (сокращение от _else if_). Программа проверяет такую цепочку сверху вниз: как только одно из условий `elif` оказывается истинным, его блок кода выполняется, а все остальные проверки ниже полностью игнорируются.
