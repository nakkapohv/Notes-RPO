#Sets #Наборы #Множества 
### Введение

Множества (`set`) - это изменяемые коллекции, состоящие из уникальных и неупорядоченных элементов. Главные характеристики этой структуры данных включают автоматическую фильтрацию дубликатов - любое повторное добавление одного и того же значения просто игнорируется - и отсутствие фиксированного порядка, из-за чего к элементам невозможно обратиться по индексу. Важно учитывать, что хотя само множество можно изменять (добавлять или удалять данные), хранить в нем можно только неизменяемые типы объектов. Поэтому внутри множества невозможно создать список или другое множество, что делает эту структуру идеальной для задач, где важна не последовательность, а наличие уникальных данных.
### Тип работы

Ключевое преимущество множеств заключается в возможности выполнения математических операций над данными, которые работают значительно быстрее, чем перебор и фильтрация обычных списков. Метод `.union()` или оператор `|` позволяют объединить все уникальные элементы нескольких наборов. Пересечение, вызываемое методом `.intersection()` или оператором `&`, оставляет только те элементы, которые есть во всех множествах сразу. Разность, через `.difference()` или `-`, возвращает элементы первого множества, которых нет во втором - при этом порядок операндов критически важен для результата. Симметричная разность (`.symmetric_difference()`) позволяет оставить все уникальные элементы, кроме общих для обоих наборов.
### Подмножества и проверка принадлежности

Создать множество можно двумя способами: перечислив элементы в фигурных скобках `{1, 2, 3}` или воспользовавшись командой `set()`. При этом важно помнить, что для создания пустого множества подходит только команда `set()`, так как фигурные скобки без содержимого Python воспримет иначе. Управлять содержимым довольно просто: `.add()` добавляет новый элемент, а `.remove()` — удаляет его (главное, не пытаться удалить то, чего нет в наборе, иначе программа выдаст ошибку). Метод `.update()` позволяет быстро объединить текущее множество с другим набором данных, `.copy()` делает независимую копию, а `.clear()` полностью очищает содержимое. На практике множества чаще всего используют, чтобы мгновенно убрать дубликаты из списков - для этого достаточно превратить список в множество и обратно через конструкцию `list(set(values))`. Также они незаменимы для быстрого сравнения больших объемов данных, фильтрации уникальных пользователей или поиска различий между массивами идентификаторов. В таких задачах множества работают значительно эффективнее и быстрее любых альтернативных способов обработки, что делает их очень полезным инструментом для программиста.

```` Python
# Список заказов, где есть повторения
orders = ["Пицца", "Бургер", "Пицца", "Суши", "Бургер"]

# Превращаем список в множество — дубликаты исчезают сами
unique_orders = set(orders)

print(unique_orders) # Выведет: {'Пицца', 'Бургер', 'Суши'}
````


```` Python
inventory = set() # Пустое множество создается только так!

inventory.add("Ключ")
inventory.add("Зелье")
inventory.add("Ключ") # Второй ключ просто проигнорируется

inventory.remove("Зелье") # Удаляем элемент

print(inventory) # Выведет: {'Ключ'}
````


```` Python
my_games = {"Doom", "Portal", "Tetris"}
friend_games = {"Portal", "Марио"}

# Пересечение (&): Игры, которые есть у нас обоих
print(my_games & friend_games) # Выведет: {'Portal'}

# Разность (-): Игры, которые есть у меня, но нет у друга
print(my_games - friend_games) # Выведет: {'Doom', 'Tetris'}
````
