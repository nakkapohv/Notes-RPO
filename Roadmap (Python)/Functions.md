#Functions #функции #def 
### Введение

Функция - это изолированный и именованный блок кода, предназначенный для выполнения конкретной задачи, который можно вызывать в программе многократно. Использование функций помогает избегать дублирования кода и делает программу структурированной и читаемой. В Python функция объявляется с помощью ключевого слова `def`, после которого идет имя функции, круглые скобки и двоеточие. Все инструкции внутри функции пишутся со строгим отступом в четыре пробела, образуя ее тело, которое выполняется только в момент вызова функции по ее имени со скобками.
### Главные преимущества использования

Создание функций опирается на фундаментальные принципы разработки. Абстракция скрывает сложную внутреннюю логику, предоставляя простой интерфейс для вызова. Инкапсуляция защищает данные: переменные внутри функции живут в изолированном пространстве имен и не конфликтуют с внешним кодом. Модульность позволяет разбивать большие программы на мелкие независимые шаги. Также функции обеспечивают повторное использование кода (принцип DRY — «не повторяйся»), повышают ремонтопригодность (ошибку нужно исправлять только в одном месте) и сильно упрощают написание тестов
### Синтаксис создания

Собственные функции определяются с помощью ключевого слова `def`, после которого пишется уникальное имя функции, обязательные круглые скобки (в них можно указать параметры-шаблоны) и двоеточие. Само тело функции пишется с отступом на следующей строке. Если функция создается «на будущее» и пока не содержит логики, в ее теле ставится оператор-заглушка `pass`, чтобы избежать синтаксической ошибки пустого блока.
### Вызов и передача аргументов

Чтобы функция отработала, ее нужно вызвать по имени с обязательными круглыми скобками (без скобок Python вернет сам объект функции, а не результат ее работы). Передавать реальные данные (аргументы) внутрь можно двумя способами:
Первый — позиционные аргументы, где переданные значения строго привязываются к параметрам по их порядку, что не всегда удобно читать. 
Второй — именованные аргументы (ключевые слова), где значения передаются в формате `имя=значение`; в этом случае порядок не важен, а код становится более понятным. 
Если способы смешиваются, позиционные аргументы обязаны идти первыми.
### Возврат значений 

Для передачи результата работы обратно в основную программу используется ключевое слово `return`, которое также немедленно завершает работу функции. Функция может возвращать любые данные, а если перечислить несколько выражений через запятую, они вернутся в виде единого кортежа. Если `return` отсутствует или написан без выражения, функция по умолчанию возвращает пустоту — объект `None`. При этом важно следить за побочными эффектами: ситуациями, когда функция напрямую меняет внешние (глобальные) объекты в памяти вместо того, чтобы вернуть новый результат. Побочных эффектов лучше избегать, так как они усложняют отладку и понимание кода.
