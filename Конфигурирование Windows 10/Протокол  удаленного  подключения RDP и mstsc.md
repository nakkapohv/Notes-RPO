#RDP #УдаленноеПодключение #mstsc 
### Введение

RDP (Remote Desktop Protocol) — это проприетарный сетевой протокол, разработанный компанией Microsoft для обеспечения удаленного доступа к рабочему столу компьютера. В отличие от простых протоколов, работающих только с пикселями (таких как RFB/VNC), RDP глубоко интегрирован в операционную систему Windows. Вместо того чтобы просто передавать картинку экрана, RDP передает графические примитивы и команды отрисовки. Это позволяет системе «собирать» изображение прямо на стороне клиента, что делает работу более плавной и экономит интернет-трафик.
### Принципы работы

Протокол работает по классической **клиент-серверной архитектуре**, но имеет важное концептуальное отличие в обработке сеансов:
- Сервер:
	Компьютер под управлением Windows (версии Pro, Enterprise или Server), к которому выполняется подключение. При входе по RDP сервер создает для пользователя **отдельную виртуальную сессию** (или перехватывает текущую, блокируя локальный экран). Локальный пользователь и удаленный пользователь не могут одновременно работать за одним экраном в десктопных версиях Windows.
- Клиент: 
	Устройство, с которого происходит управление. Клиент получает команды отрисовки, воспроизводит звук с удаленного ПК и отправляет обратно команды с клавиатуры и мыши.

RDP позволяет «пробрасывать» устройства клиента на сервер. Вы можете подключиться к удаленному ПК и использовать на нем свой локальный принтер, обмениваться файлами через общий буфер обмена или видеть свои локальные жесткие диски в удаленной системе.
### Особенности встроенной утилиты mstsc

mstsc (Microsoft Terminal Services Client) — это официальная встроенная утилита Windows для работы с RDP. Ее не нужно скачивать или устанавливать дополнительно.

Ключевые преимущества использования mstsc:

- Оптимизация: Утилита умеет динамически адаптировать качество графики (отключать тени, анимации, менять глубину цвета) в зависимости от скорости интернет-соединения.
- Кэш: Программа кэширует часто повторяющиеся элементы интерфейса (например, иконки, элементы окон) на стороне клиента, чтобы не передавать их по сети дважды.
- Удобство развертывания: Файлы настроек подключения можно сохранять в формате `.rdp` и передавать другим пользователям для быстрого запуска в один клик.
### Настройка подключения через mstsc

Для успешного соединения необходимо предварительно настроить принимающий компьютер (Сервер) и запустить утилиту на управляющем (Клиент).
#### Настройка Сервера 

1. **Проверка версии ОС:** Убедитесь, что на целевом компьютере установлена Windows версии Pro (домашние версии Windows Home не могут выступать в роли RDP-сервера).
2. **Включение функции:** Перейдите в «Параметры» ➔ «Система» ➔ «Удаленный рабочий стол» и переведите переключатель в положение «Вкл».
3. **Учетная запись:** Убедитесь, что ваша учетная запись Windows имеет пароль (RDP по умолчанию запрещает подключение пользователям без пароля).
4. **Идентификация:** Узнайте IP-адрес этого ПК в локальной сети (например, `192.168.1.15`) или его точное сетевое имя.
#### Настройка Клиента 

1. **Запуск утилиты:** Нажмите комбинацию клавиш `Win + R`, введите команду `mstsc` и нажмите Enter (или найдите программу «Подключение к удаленному рабочему столу» через поиск в меню Пуск).
2. **Настройка параметров (опционально):** В появившемся окне нажмите «Показать параметры». Здесь вы можете настроить разрешение экрана, проброс буфера обмена и локальных дисков (вкладка «Локальные ресурсы»).
3. **Ввод адреса:** На вкладке «Общие» в поле _Компьютер_ введите IP-адрес или имя сервера (`192.168.1.15`), а в поле _Пользователь_ — логин учетной записи удаленного ПК.
4. **Авторизация:** Нажмите **Подключить**. Система запросит пароль от учетной записи Windows на удаленном ПК. После проверки откроется окно с полноценным рабочим столом.
